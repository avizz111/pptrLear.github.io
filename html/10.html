Curso 10: Buenas Prácticas en HTML

Aprende cómo escribir HTML limpio, accesible y mantenible.

¿Por qué seguir buenas prácticas?

Escribir código HTML con buenas prácticas mejora la accesibilidad, el rendimiento, la legibilidad y la colaboración en equipo.

1. Usa etiquetas semánticas

Prefiere <header>, <section>, <article>, <footer> sobre <div> genéricos.

<article>
  <h2>Título del artículo</h2>
  <p>Contenido...</p>
</article>

2. Indenta correctamente

La indentación mejora la legibilidad y evita errores.

3. Usa el atributo alt en imágenes

Ayuda a usuarios con lectores de pantalla y mejora el SEO.

4. Evita contenido obsoleto

No uses etiquetas como <font> o <center>; usa CSS.

5. Valida tu HTML

Usa herramientas como el validador de W3C para asegurarte de que tu código es correcto.

6. Escribe atributos con comillas

Siempre encierra los valores de atributos entre comillas.

7. Usa comentarios solo cuando sea necesario

<!-- Esta sección muestra los artículos principales -->

8. Organiza tus archivos

Separa HTML, CSS y JS en carpetas como css/, js/ y img/.
